WeChat Mini Program

把每日行动，养成一颗会生长的星球。

星途自律舱不是普通打卡清单。它把运动、饮食、学习、工作、计划、睡眠六类行动 连接到星球生态：每一次完成都会获得星光，点亮区域，推动荒芜星球逐渐复苏。

Product Idea

先让用户有目标，再让打卡有反馈。

项目设计的重点不是把按钮做多，而是补齐完整体验链路：目标设定、计划生成、 当日行动、即时奖励、生态变化、数据复盘。用户完成的不是一次孤立打卡， 而是在建设自己的星球。

Feature System

核心功能模块

围绕真实自律场景，把“想做什么、今天做什么、做完有什么变化”串起来。

01

启动与命名

首次进入先启动星球，再给星球命名，建立用户和星球之间的拥有感。

02

目标选择

用户选择均衡自律、学习提升、健身塑形、项目冲刺等节奏，系统推荐今日行动。

03

自定义计划

用户可以添加自己的计划，选择六类维度、行动内容和提醒节奏，保存后用于日常打卡。

04

六维打卡

运动、饮食、学习、工作、计划、睡眠分别对应不同生态区和成长反馈。

05

星光奖励

打卡后立即显示星光结算和本次点亮内容，避免用户完成后没有感知。

06

宝箱与成就

每日六项完成后领取宝箱，长期行动沉淀为连续打卡、生态解锁和星光累计成就。

Growth Loop

从荒芜星到生态星

星球不是装饰图，而是项目的反馈系统。初始状态是一颗荒芜星， 用户完成行动后逐步出现光点、植被、建筑和生态区入口。

Implementation

技术与工程设计

小程序原生开发

使用 JavaScript、WXML、WXSS 组织页面、组件和交互逻辑。

微信云开发

通过云函数处理登录、打卡提交、统计读取、资料更新、数据导出和云资源链接。

本地优先数据

本地缓存保证基础体验，云端同步用于多次打开后的数据恢复。

包体控制

大视频、原图和源素材走云存储，包内只保留压缩后的必要资源。

测试覆盖

核心计算逻辑使用 Node.js Test Runner 覆盖日期、目标、奖励、成就和统计。

文档整理

保留运行说明、云开发配置和资源规范，便于后续继续迭代。
