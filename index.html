Starlight discipline

星途自律舱不是普通打卡清单。它把运动、饮食、学习、工作、计划、睡眠六类日常行动， 转化为星光、宝箱、成就徽章和生态区成长，让自律从“点一下完成”变成能被看见的持续建设。

Product Idea

先建立目标，再让每次打卡都有反馈。

项目围绕“目标设定 → 计划生成 → 当日行动 → 即时奖励 → 生态变化 → 数据复盘”构建闭环。 用户完成的不再是一条孤立任务，而是在亲手修复自己的星球：荒芜表面会逐渐出现河流、花园、建筑和光点。

Feature System

核心功能模块

从首次进入到长期复盘，项目用完整体验链路增强用户持续打卡动力。

01

星球启动与命名

首次进入先播放启动动画，再进入星球命名，让用户对自己的成长星球建立拥有感。

02

目标与计划

提供均衡自律、学习提升、健身塑形、项目冲刺等目标，也支持用户自定义六类打卡计划。

03

六维打卡

运动、饮食、学习、工作、计划、睡眠分别对应生态区成长，避免所有打卡都只是同一个按钮。

04

奖励结算

完成后弹出星光结算，告诉用户获得了什么、点亮了哪里、距离宝箱还有多远。

05

宝箱与成就

每日六项完成后可领取宝箱；连续打卡、首次点亮、完美一天等行为会沉淀为徽章墙。

06

数据复盘

热力图、趋势线、连续天数、星光值和完成率帮助用户看到自己的节奏，而不是只看当天。

Growth Loop

从荒芜星到生态星，打卡驱动 8 级进化。

星球不是装饰图，而是项目的核心反馈系统。初始状态是一颗荒芜星球，用户每完成一次行动都会获得星光。 星光累计后，星球从萌芽星、绿洲星、山河星逐步进化到文明星、星辉星、卫星星和光环星。

Six Ecology Zones

六大生态区

每一个维度都对应一片可进入的生态区，长期坚持会解锁建筑、装饰和光效。

Implementation

技术与工程设计

原生小程序

使用 JavaScript、WXML、WXSS 组织页面和交互，适配微信开发者工具与真机运行。

微信云开发

云函数覆盖登录、打卡提交、统计读取、资料更新、数据导出和云资源链接。

本地优先

本地 Storage 保证即时响应，云端同步用于数据备份和跨设备恢复。

资源分层

大视频和原图走云存储，本地包保留压缩后的必要资源，降低小程序包体压力。

测试覆盖

使用 Node.js Test Runner 覆盖奖励计算、成就解锁、目标推荐和统计聚合等核心逻辑。

可展示性

项目同时整理 README、云开发说明和素材说明，便于作为求职作品展示。
