Product Idea

让“自律”不再像任务表，而像一场星球复苏。

传统打卡容易变成机械勾选，完成后反馈弱、动力衰减快。这个项目把每日行动设计成「目标设定 - 行动执行 - 即时奖励 - 生态成长 - 数据复盘」的闭环，让用户看见自己的投入正在改变一个可视化世界。

01 先定目标

首页根据目标推荐今日行动，避免用户一进来不知道做什么。

02 进入行动舱

学习、工作、运动等行动进入计时场景，记录真实投入。

03 获得反馈

完成后立即弹出星光、宝箱、生态解锁与成就反馈。

User Journey

完整使用闭环

1

命名星球

首次进入完成启动动画和星球命名，建立个人归属感。

2

制定计划

自定义行动内容、类别、目标量和提醒时间。

3

行动计时

进入行动舱，实时显示投入时长并记录完成结果。

4

结算奖励

弹窗展示星光、宝箱进度、成就和生态变化。

5

复盘成长

通过热力图、趋势数据和生态区查看持续成长。

Core Modules

功能不是堆上去，而是围绕打卡动力设计。

星球总览

展示日期、星光、今日进度、目标状态和生态入口。星球会随累计行动从荒芜逐步复苏。

即时奖励

打卡完成后直接弹出结算，不让用户滑动寻找反馈。

星光成长

每次行动获得星光值，推动星球等级、生态区和成就成长。

Action Cabin 行动舱计时

适合学习自习、工作专注、运动训练、睡前整理等真实场景。开始、计时、完成、记录形成闭环。

Six Eco Zones

六类习惯，对应六个生态区。

Growth System

成长不是一个数字，而是一组可见变化。

星光值、连续天数、六维打卡、完美日、行动舱记录共同驱动星球成长。用户可以在首页看到总览，也可以进入生态区查看具体建筑和装饰变化。

  • 八级星球成长体系
  • 六维生态解锁进度
  • 宝箱、星光币、成就徽章
  • 热力图和趋势数据复盘

Tech Stack

原生微信小程序 + 云开发。

WXML / WXSS / JS 原生小程序前端

多页面结构、组件化拆分、工具层封装，适配小程序运行环境。

Cloud Functions 微信云函数

登录、打卡提交、统计看板、资料更新、数据导出、云资源链接。

Local First 本地优先数据

Storage 即时反馈，云端同步兜底，离线或网络波动时仍能使用。

Canvas / Assets / Audio 视觉与声音反馈

星球画面、生态图片、奖励图标、音效反馈共同强化完成感。
